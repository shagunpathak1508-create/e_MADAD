# eमदद (eMADAD) — Emergency Response App

> Orange · Green · White — India Tricolour Emergency Network

---

## 🚀 Step-by-Step Setup Guide (Read This First!)

Follow these steps **in order**. Each step links back to others.

---

### STEP 1 — Install Required Tools

Open **PowerShell** or **Command Prompt** and check what you have:

```powershell
flutter --version   # Should show Flutter 3.x
node --version      # Should show v18+
```

If Flutter is missing → https://docs.flutter.dev/get-started/install/windows

---

### STEP 2 — Create a Firebase Project

1. Go to **https://console.firebase.google.com**
2. Click **"Add project"** → name it `emadad`
3. Disable Google Analytics (optional) → **Create project**
4. Once created, click **"Build"** → **"Firestore Database"**
   - Click "Create database" → Start in **test mode** → Choose any region → Done
5. Click **"Build"** → **"Authentication"**
   - Click "Get started" → Enable **Anonymous** sign-in → Save

---

### STEP 3 — Install Firebase CLI Tools

```powershell
# Install Firebase CLI
npm install -g firebase-tools

# Install FlutterFire CLI
dart pub global activate flutterfire_cli
```

---

### STEP 4 — Connect Firebase to the App

```powershell
# Navigate to project folder
cd C:\Users\123456\Desktop\eMADAD

# Login to Firebase
firebase login

# Run FlutterFire configuration — follow the prompts
# Select your "emadad" project when asked
flutterfire configure
```

> ✅ This auto-generates `lib/firebase_options.dart` with your real API keys.
> It **replaces** the placeholder file that already exists.

---

### STEP 5 — Set Up Firestore Security Rules

In Firebase Console → **Firestore** → **Rules** tab, paste:

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /volunteers/{doc} {
      allow read, write: if request.auth != null;
    }
    match /help_requests/{doc} {
      allow read, write: if request.auth != null;
    }
  }
}
```
Click **Publish**.

---

### STEP 6 — Run the App

```powershell
cd C:\Users\123456\Desktop\eMADAD

# Get all dependencies
flutter pub get

# Connect your Android phone (enable Developer Mode + USB Debugging)
# OR start Android Emulator in Android Studio

# Run the app
flutter run
```

---

### STEP 7 — App Flow (After Launch)

```
Launch
  └─► Permissions Screen (Grant Location + Contacts)
        └─► Home Screen
              ├─► "I Need Help" (orange)

                    └─► Returning  → Volunteer Dashboard
```

## 🗺️ Map Setup

The app uses **OpenStreetMap + flutter_map** — completely free, no API key required.

Nearby hospitals, shelters, police stations, and fire departments are fetched using the **Overpass API** (free, open data).

---

## 📡 Offline Mode

When a device loses connectivity:
- Last known GPS location is stored locally (SharedPreferences)
- Movement trail (last 20 points) is shown on the map
- Direction of travel is predicted from the last 2 GPS points
- A red banner appears: **"User is offline · Last updated X min ago"**
- Emergency SMS can still be sent (SMS works even on congested networks)

## ⚠️ Common Issues

| Problem | Fix |
|---------|-----|
| `firebase_options.dart` errors | Run `flutterfire configure` to replace the placeholder |
| Location not working | Enable Location in phone Settings → Apps → eMADAD |
| Map not loading | Check internet — OpenStreetMap requires a connection |
| SMS not opening | Check SEND_SMS permission in phone Settings |
| Build fails with minSdk error | Already fixed to minSdk 21 in `android/app/build.gradle.kts` |
